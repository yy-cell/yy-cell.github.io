OSI参考七层模型：
应用层：提供用户界面
表示层：保护数据进行加密等 ：数据加密压缩转换服务
会话层：将不同应用程序的数据分离 ：对话控制
传输层：提供可靠或者不可靠的传输，在重传前进行纠错（tcp协议udp协议）：端到端连接 数据表示为字段
网络层：提供逻辑地址，路由器使用他们来选择路径（IP协议） ：路由选择 数据表示为组
数据链路层：将分组拆分为字节，并将字节组合成为帧，使用MAC地址提供介质错误检测，但不纠错（交换机）（ppp协议点对点协议，fr帧中继协议） ：成帧
物理层：在设备之间传输比特，指定电平电缆速度和电缆针脚（集线器） ：物理拓扑 数据表示为比特

1、数据的封装与解封装
上层数据
TCP报头（源端口 目标端口）上层数据
IP报头（源IP目标IP）TCP报头（源端口 目标端口）上层数据
LLC报头 IP报头（源IP目标IP）TCP报头（源端口 目标端口）上层数据
MAC报头 LLC报头（llc 报头是数据封装时在数据链路层的附加标识） IP报头（源IP目标IP）TCP报头（源端口 目标端口）上层数据
TCP:提供可靠的连接服务，采用三次握手建立连接：
第一个是“连接协定”数据段，用于请求同步
第二个接下来的数据段确认请求，并且在主机之间确定连接参数（即规则）。这些数据段也请求同步接收方的排序，以建立双向连接
第三：最后一个也是进行确认，通知目标主机连接协定已被接受且连接已建立
第一：发送方SYN
第二：接收方SYN/ACK
第三：发送方ACK
连接已建立
网络架构三层模型
核心层：唯一目标是尽可能快的交换数据流 （少做限制）
汇聚层：进行网络互连
接入层：
DOD模型：是OSI模型的精简版，只有四层 TCP/IP四层模型
进程/应用层： （是七层模型中的上三层） 应用层
主机到主机层：（为七层osi模型的传输层） 运输层
因特网层： （为七层模型的中的网络层） 网际层
网络接入层：（为七层模型中的数据链路层，和物理层） 网络接口层
DOD模型：各层对应的协议
进程/应用层： Telent FTP LPD SNMP
TFTP SMTP NFS XWindow
主机到主机层：TCP UDP
因特网层： ICMP ARP RARP IP
网络接入层： 以太网 快速以太网 令牌环 FDDI
Telent:专司终端模拟，远程访问协议，本质就是让远程客户端看起来根与本地直接连接的终端相似
FTP:文件传输协议，既是协议，可供程序使用，作为程序，可直接手工执行与文件相关的任务，可用anonymous避开身份验证
TFTP:是FTP的简化版,使用简单速度快 功能没FTP丰富，没有提供目录浏览功能，除了接受发送文件外啥也不能做，不需要身份验证，所有不安全，支持网站少
NFS:是一种致力于文件共享的协议（及不同的系统之间可以访问相同的文件）
SMTP:简单邮件传输协议 用于发送电子邮件，而POP3或者IMAP 用于接收文件
POP：邮局协议 提供了一种对到来邮件进行存储的机制
IMAP4:因特网消息访问协议，让你能够控制邮件的下载方式，因此使用它可获得亟需的安全性
TLS：TLS)及其前身 SSL都是加密协议，非常适合用于确保在线数据传输的安全
SIP:是一种非常流行的信令胁议，用于建立和拆除多媒体通信会话
RTP:是一种分组格式标准，用于通过因特网传输语音和视频
LDP:用于共享打印机,
X Window:为客户端/服务器操作设计的 是一种编写基于 GUI(图形用户界面)的客户端/服务器应用程序的协议。其基本思想是，让运行在 台计算机上的客户端程序能
够通过窗口服务器显示另一台计算机的内容
SNMP：简单网络管理协议，收集并操作有价值的网络信息
SSH：安全外壳 (SSH) 协议通过标准 TCPIIP 连接建立安全的 Telnet 会话，用于执行如下操作:登录系统、在远程系统中运行程序以及在系统间传输文件等。
HTTP：HTTP超文本传输协议用于管理 Web 浏览器和 Web 服务器之间的通信
HTTPS：安全超文本传输协议，使用 SSL（安全套接字层）
NTP:用于将计算机时钟与标准时间源(通常是原子钟)同步
NNTP:网络新闻传输协议
SCP；安全复制协议，首先在发送主机和接收主机之间建立一条安全的加密连接，并一直保持这种状态，直到文件传输完毕
LDAP：轻量级目录访问协议
IGMP：因特网组管理协议，是一种用于管理 IP 组播会话的TCP/IP 协议
LPR：LPR (行式打印机)和 LPD (Line Printer Daemon ，行式打印机守护进程)来完成打印作业。
DNS：域名服务，解析主机名
DHCP/BootP：动态主机配置协议，给主机分配 IP 地址
DHCP BootP差别在于， BootP 给主机分配田地址，但必须手工将主机的硬件地址输入到 BootP表中。你可将 DHCP 视为动态的 BootP
BootP也可用于发送操作系统，让主机使用它启动，而 DHCP 没有这样的功能。
DHCP 服务器和客户端之间的4次交互：
(1) DHCP 客户端广播一条 DHCP 发现消息，旨在寻找 DHCP 服务器(端口 67 );
(2) 收到 DHCP 发现消息的 DHCP 服务器向主机发回一条单播 DHCP 提议消息;
(3) 客户端向服务器广播一条 DHCP 请求消息，请求提议的 IP 地址和其他信息:
(4) 服务器以单播方式发回一条 DHCP 确认消息，完成交互。
DHCP 冲突：就是两台主机使用了相同的IP地址
1、DHCP服务器ping地址池中的IP如果没有应答，那么DHCP 就认为该地址未被分配
2、主机也可以广播自己的地址
3、主机也可以使用免费ARP来帮助避免地址重复，DHCP 客户端在本地 LAN或者VLAN 中发送ARP广播，并要求解析分配新给它的地址，从而将冲突消灭在萌芽状态。
APIPA:自动私有 IP编址,APIPA 使用的 地址范围为169.254.0.1-169.254.255.254 ，客户端还会给自己配置默认的 类子网掩码一-255.255.0.0
如果在拥有DHCP服务器的公司网络中出现了这一范围的IP地址，那么说明该DHCP客户端不正常，或者是服务器停止运行或者网络问题不可达。
主机到主机层协议：tcp udp
tcp 将来自应用程序的大型数据块经行划分成为数据段，并且将其标上编号，让接收者按照顺序排列数据块，完成传送后，发送者会等待接收者的确认后，并重新传送未的到确认的数据段
tcp创建的虚电路即是三次握手，面向连接。
TCP:是一种可靠的精确协议，采用全双工模式，面向连接。
UDP：就是TCP的一个简化版，适用于不需要可靠传输的信息
不建立虚电路，也成为无连接协议
TCP与UDP区别
TCP UDP
排序 不排序
可靠 不可靠
面向连接 无连接
虚电路 低开销
确认 不确认
使用窗口技术控制流量 不使用窗口技术或其他流量控制方式
使用tcp的服务：FTP HTTP Telent SMTP
使用udp的服务：DHCP SNMP TFTP
TCP UDP 必须使用端口号才能与上层通信 （端口号小于1024的为知名端口号）
常见应用层端口号：
TCP: Telent 23; SMTP 25;HTTP 80;FTP20 21;DNS 53;HTTTPS 443;SSH 22;POP3 110;NTP 123;IMQP4 143;
UDP:SNMP 161;TFTP 69;BootPS/DHCP 67
